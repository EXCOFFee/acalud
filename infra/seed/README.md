# Seed (datos ficticios de demostración)

Datos **100 % ficticios** (R-04) que se **diseñan como si fueran reales**: CUIT con dígito
verificador válido, emails, domicilios AR plausibles. El seed de demo (NFR-C1/C3) es un
superconjunto reproducible usado también por el load test (200 cuentas, 20 instituciones,
100 juegos, 5 000 pedidos, 10 000 sesiones de uso).

> Estado: se construye a partir de la **Etapa 1** (esqueleto E2E) y se completa en la Etapa 2.
